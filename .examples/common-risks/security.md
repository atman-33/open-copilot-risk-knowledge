# セキュリティリスク

すべての機能で発生する可能性のある一般的なセキュリティリスク。

## 1. SQLインジェクション

- **リスク**: 攻撃者がデータベースクエリを操作して、権限のないデータにアクセス・変更できる。
- **原因**:
  - SQLクエリでの直接的な文字列連結
  - 入力値検証の不十分さ
- **重要度**: 重大（データ漏洩、システム侵害）

## 2. クロスサイトスクリプティング（XSS）

- **リスク**: 悪意のあるスクリプトがユーザーのブラウザに注入・実行される。
- **原因**:
  - HTML出力でのエスケープされていないユーザー入力
  - リッチテキストコンテンツの不適切な処理
- **重要度**: 高（セッションハイジャック、データ盗難）

## 3. 情報開示

- **リスク**: エラーメッセージ、ログ、またはAPIレスポンスを通じて機密データが露出。
- **原因**:
  - 本番環境での冗細なエラーメッセージ
  - 機密データ（パスワード、トークン、個人情報）のログ
  - 不要なフィールドを含むAPIレスポンス
- **重要度**: 高（プライバシー違反、規制違反）

## 4. 認証バイパス

- **リスク**: 攻撃者が適切な認証なしで保護されたリソースにアクセスできる。
- **原因**:
  - 弱いセッション管理
  - 不適切なトークン検証
  - エンドポイントの認証チェック欠落
- **重要度**: 重大（不正アクセス）

## 5. 不正なリソースへの直接参照（IDOR）

- **リスク**: ユーザーが識別子を操作することで他のユーザーのデータにアクセスできる。
- **原因**:
  - リソースアクセスの認可チェック欠落
  - 予測可能なリソースID
- **重要度**: 高（データ漏洩）
