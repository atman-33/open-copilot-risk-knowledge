# 可用性リスク

システムの継続的な運用を脅かす一般的な可用性リスク。

## 1. ログイン/認証失敗

- **リスク**: ユーザーがシステムにログインできない。
- **原因**:
  - 認証APIのレスポンス形式変更によるフロントエンド破損
  - セッションストア（Redisなど）の接続失敗
  - 証明書の失効
- **重要度**: 重大（ビジネス全体の停止）

## 2. データベース接続枯渇

- **リスク**: コネクションプールの枯渇により新規リクエストが処理できない。
- **原因**:
  - データベーストランザクション内での外部API呼び出し
  - コネクションリーク
- **重要度**: 重大（サービス停止）

## 3. 外部サービス依存の障害

- **リスク**: 依存サービスが失敗するとアプリケーションが利用不可になる。
- **原因**:
  - 外部API障害への代替メカニズムがない
  - サーキットブレーカーパターンの欠落
  - 非重要サービスへの同期依存
- **重要度**: 高～重大（サービスの重要度に依存）

## 4. デプロイ関連の停止

- **リスク**: デプロイ中または後にシステムが利用不可になる。
- **原因**:
  - データベースマイグレーション失敗
  - 設定エラー
  - 互換性のないAPIバージョンのデプロイ
- **重要度**: 重大（サービス中断）

## 5. リソース枯渇

- **リスク**: リソース枯渇によりシステムが応答しなくなる。
- **原因**:
  - ディスク領域枯渇（ログ、一時ファイル）
  - メモリリーク
  - スレッドプール枯渇
- **重要度**: 重大（サービス停止）
